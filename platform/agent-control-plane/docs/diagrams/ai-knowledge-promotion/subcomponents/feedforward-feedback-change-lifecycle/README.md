# Feedforward and Feedback Change Lifecycle

This diagram is the detailed view of the feedforward/feedback interface in the
parent [AI knowledge promotion diagram](../../README.md).

The parent diagram treats the lifecycle as a system boundary:

- governed context feeds forward into engineering work;
- proposed changes, validation results, review findings, incidents, lessons,
  and retrieval performance feed back into knowledge capture and promotion.

This subcomponent expands what happens inside that boundary: context loading,
artifact generation, automated self-correction, human review, the promotion
decision, and validation through the harness pipeline.
